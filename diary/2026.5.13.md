# 本次 Git + ROS2 踩坑完整总结（最终完整版，新增 `git push` 补充）
## 一、核心网络问题：HTTPS 443端口被墙 → 改用SSH协议（22端口）
1. **问题现象**
   - 旧仓库用 `https://github.com/xxx.git`（HTTPS协议，443端口）
   - `git push` / 同步一直报：`Failed to connect to github.com port 443: Connection refused` 连接被拒绝
   - 新仓库用 `git@github.com:xxx.git`（SSH协议，22端口），**直接正常同步**
2. **解决方法**
   - 仓库地址切换为SSH：`git remote set-url origin git@github.com:用户名/仓库名.git`
   - 全局清理失效镜像（`ghproxy`）：`git config --global --unset url."https://mirror.ghproxy.com/".insteadOf`
3. **原理**：国内网络拦截GitHub的**443端口**，**22端口通常正常**，SSH协议完美规避

## 二、ROS2项目Git规范：**只提交src，不提交整个工作空间**
1. **问题现象**
   - 把工作空间整体上传，把`build/`、`install/`、`log/`编译缓存、日志文件一起提交
   - Git修改列表出现海量垃圾文件、推送极慢、文件冗余
2. **正确规范**
   - ROS2项目：**只上传`src`里的代码包**，`build/install/log`是本地编译产物，**绝对不提交**
   - 别人下载：自行新建工作空间，把src放入后编译
3. **解决方法**
   - 本地取消追踪缓存文件：`git rm -r --cached build install log`
   - 新建`.gitignore`忽略：永久屏蔽缓存文件，不再显示在修改列表

## 三、Git核心命令：`git pull`、`git push`、提交消息弹窗区别
1. **`git pull`**：**拉取远程（GitHub网页端/他人）最新代码，同步到本地**
   - 作用：网页端修改内容后，本地执行`git pull`，本地自动更新为网页端最新版
2. **`git push`**：**把本地已提交的修改，上传同步到GitHub网页端**
   - 关键：**`git push 不会弹出提交消息框`**
3. **提交消息弹窗来源**
   - 只有 `git commit`（**本地提交**）才会弹出编辑框，要求填写修改备注；
   - `git push` 只做上传动作，不涉及填写备注。
4. 标准完整流程：本地修改 → `git commit`填写备注 → `git push`上传；网页端修改 → `git pull`拉取更新

## 四、VSCode Git日志疯狂刷屏问题
1. **问题现象**：输出面板疯狂输出`not a git repository`报错
2. **原因**：VSCode同时打开了**多个文件夹**，Git插件扫描了非Git仓库目录
3. **解决**：只打开**单个Git仓库文件夹**，或关闭VSCode自动Git扫描

# 最终记住4条铁律
1. 以后GitHub仓库**统一用SSH地址**，彻底解决443端口连接问题
2. ROS2项目Git只传**src源代码**，不传工作空间和编译文件
3. 远程更新用`git pull`，本地上传用`git push`
